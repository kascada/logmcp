# LogMCP — Configuration Reference

Location: `/etc/logmcp/config.yaml`

Generated by `logmcp setup`. Can be edited directly; restart the service afterwards:

```bash
sudo systemctl restart logmcp
logmcp check
```

---

## Full Example

```yaml
name: "logmcp-myserver"

server:
  host: "127.0.0.1"
  port: 7788
  tls:
    mode: "off"

proxy:
  enabled: true
  trusted_proxy: true
  domain: "example.com"
  path_prefix: "/logmcp"
  caddy: true

auth:
  tokens:
    - name: "claude-code"
      token: "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
      scopes: ["read"]

logs:
  whitelist:
    - "/var/log/*"
    - "/var/log/asterisk/*"
  blacklist:
    - "/var/log/private/**"
  journald: false

audit:
  syslog: true

tools:
  disabled: []

extensions:
  switchboard:
    enabled: false
    calls_dir:       "/var/log/switchboard/calls"
    sim_dir:         "/var/log/switchboard/sim"
    transcripts_dir: "/var/log/switchboard/transcripts"
  databases:
    mysql:
      - name: "mydb"
        dsn:  "user:pass@tcp(host:3306)/dbname"
```

---

## `name`

| Type | Default |
|------|---------|
| string | `logmcp-<hostname>` |

Human-readable server name. Shown to AI clients in the MCP server list. Defaults to `logmcp-` + the system hostname if not set.

---

## `server`

### `server.host`

| Type | Default |
|------|---------|
| string | `0.0.0.0` |

Bind address. Set to `127.0.0.1` when running behind a reverse proxy.

### `server.port`

| Type | Default |
|------|---------|
| int | `7788` |

TCP port logmcp listens on. `logmcp check` verifies the port is available.

### `server.tls.mode`

| Type | Values | Default |
|------|--------|---------|
| string | `self-signed` \| `custom` \| `off` | `self-signed` |

| Mode | When to use |
|------|-------------|
| `self-signed` | logmcp manages a self-signed certificate; AI client must trust it |
| `custom` | Provide your own certificate and key |
| `off` | Plain HTTP; use only behind a TLS-terminating reverse proxy |

### `server.tls.cert` / `server.tls.key`

| Type | Default |
|------|---------|
| string | `/etc/logmcp/server.crt` / `/etc/logmcp/server.key` |

Paths to the TLS certificate and private key. Only used when `tls.mode` is `custom` or `self-signed`. Certificates are generated by `logmcp setup`, never by the running server.

---

## `proxy`

### `proxy.enabled`

| Type | Default |
|------|---------|
| bool | `false` |

Set to `true` when logmcp runs behind a reverse proxy (Caddy, nginx, etc.).

### `proxy.trusted_proxy`

| Type | Default |
|------|---------|
| bool | `false` |

Read the real client IP from `X-Forwarded-For` for audit log entries. Only enable when logmcp is exclusively reachable via the reverse proxy — otherwise the header can be forged.

### `proxy.domain`

| Type | Default |
|------|---------|
| string | `""` |

Public hostname (e.g. `example.com`). Required when `proxy.enabled: true`. Used in `client-config` snippets and Caddyfile output.

### `proxy.path_prefix`

| Type | Default |
|------|---------|
| string | `""` |

URL subpath (e.g. `/logmcp`). Empty means serve at the domain root. Used in `client-config` snippets.

### `proxy.caddy`

| Type | Default |
|------|---------|
| bool | `true` |

Generate Caddy-specific snippets in `logmcp setup` and `logmcp service caddy-snippet`.

---

## `auth`

### `auth.tokens[]`

List of bearer tokens. At least one entry is required.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | **Yes** | Human-readable label; shown in audit log entries |
| `token` | string | **Yes** | Bearer token value presented by clients in `Authorization: Bearer` |
| `scopes` | list | **Yes** | Currently only `"read"` is defined |

Multiple tokens allow per-client revocation without invalidating all clients:

```yaml
auth:
  tokens:
    - name: "claude-code"
      token: "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
      scopes: ["read"]
    - name: "vscode"
      token: "yyyyyyyy-yyyy-yyyy-yyyy-yyyyyyyyyyyy"
      scopes: ["read"]
```

**Legacy migration:** configs with the old `auth.token: "<value>"` single-field format are automatically migrated on load to a token named `"default"` with scope `["read"]`. Run `logmcp setup` to persist the new format.

Token management without manual YAML editing: `logmcp token list|add|remove|renew`.

## `logs`

### `logs.whitelist`

| Type | Default |
|------|---------|
| list of glob strings | `["/var/log/*"]` |

Glob patterns for accessible log files. Only files matching a whitelist pattern (and not a blacklist pattern) are accessible.

If the list is empty or the key is absent from the config, the default `["/var/log/*"]` is used automatically. The generated config file includes additional common paths as commented-out examples.

Pattern semantics:
- `*` — matches within a single directory level
- `**` — matches any number of path components (prefix match)
- Paths are canonicalized (symlinks resolved, `..` eliminated) before matching
- Matched directories are silently skipped — only regular files are served

### `logs.blacklist`

| Type | Default |
|------|---------|
| list of glob strings | `[]` |

Glob patterns that deny access regardless of whitelist matches. Blacklist takes precedence.

### `logs.journald`

| Type | Default |
|------|---------|
| bool | `false` |

Expose the systemd journal as a virtual log source. Appears in `list_logs` as `journald://`. Use `journald://unit.service` to filter to a single unit. Reads via `journalctl --no-pager --output=short-iso`.

---

## `audit`

### `audit.syslog`

| Type | Default |
|------|---------|
| bool | `true` |

Write every MCP tool call and denied access to syslog. Log content and search patterns are redacted. View with `grep logmcp /var/log/syslog` or `journalctl -t logmcp`.

---

## `tools`

### `tools.disabled`

| Type | Default |
|------|---------|
| list of strings | `[]` |

MCP tool names to hide from AI clients. Useful for restricting which tools a particular deployment exposes. Valid values: `list_logs`, `read_log`, `search_log`, `log_info`, `check_environment`, `switchboard_debug`.

Example — expose only listing and reading, no search:

```yaml
tools:
  disabled:
    - search_log
```

---

## `extensions`

### `extensions.switchboard`

Optional Switchboard extension for call-center log analysis.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | bool | `false` | Enable the extension |
| `calls_dir` | string | `/var/log/switchboard/calls` | Call log directory |
| `sim_dir` | string | `/var/log/switchboard/sim` | Simulation log directory |
| `transcripts_dir` | string | `/var/log/switchboard/transcripts` | Transcript directory |

When enabled, exposes the `switchboard_debug` MCP tool.

### `extensions.databases.mysql[]`

Optional list of MySQL connections. Used by `logmcp check` (TCP reachability) and the `switchboard_debug` tool.

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Human-readable label shown in `logmcp check` output |
| `dsn` | string | Go sql-driver DSN: `user:pass@tcp(host:port)/dbname` |

Not written by `logmcp setup` — add manually.
